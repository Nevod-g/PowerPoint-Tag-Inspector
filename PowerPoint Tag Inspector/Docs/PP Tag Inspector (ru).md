Разработай desktop-приложение для MS Windows на базе библиотеки `Miramac/node-office-script`.

Цель приложения: дать пользователю простой UI для просмотра и редактирования PowerPoint Tags у активного слайда или выбранной фигуры в текущей активной презентации PowerPoint.

## Технологическая база
Developer stack: C# WinForms + PowerPoint COM Interop

Использовать:
* Node.js
* `Miramac/node-office-script`
* Desktop UI framework на выбор:

  * Electron предпочтительно;
  * допускается другой desktop wrapper, если он лучше подходит.
* Приложение должно работать с уже открытым экземпляром Microsoft PowerPoint.
* Приложение должно подключаться к активной презентации PowerPoint через `node-office-script`.
* Не создавать новую презентацию без явной команды пользователя.
* Основной сценарий — работа с текущей активной презентацией, активным слайдом и выбранной фигурой.

## Основной UI

Окно приложения должно содержать две основные секции:

### 1. Options

В секции Options должен быть выбор источника Tags:

1. `presentation.activeSlide()`
2. `presentation.selectedShape()`

UI можно реализовать через radio buttons:

* Active Slide
* Selected Shape

При переключении опции приложение должно загружать Tags соответствующего объекта.

### 2. Tags

В секции Tags должна отображаться таблица со списком тегов выбранного объекта.

Таблица должна иметь колонки:

* Name
* Value

Для списка Tags должны быть доступны операции:

* Add Tag
* Edit Tag
* Delete Tag
* Refresh

## Поведение приложения

### Инициализация

При запуске приложение должно:

1. Подключиться к активному экземпляру PowerPoint.
2. Получить активную презентацию.
3. По умолчанию выбрать режим `presentation.activeSlide()`.
4. Получить активный слайд.
5. Загрузить и отобразить Tags активного слайда.

Если PowerPoint не запущен, активная презентация отсутствует или активный слайд недоступен — показать понятное сообщение об ошибке в UI.

### Режим Active Slide

Когда выбран режим `presentation.activeSlide()`:

1. Получить активный слайд.
2. Прочитать все Tags слайда.
3. Отобразить Tags в таблице.
4. Операции Add/Edit/Delete должны применяться к Tags активного слайда.

### Режим Selected Shape

Когда выбран режим `presentation.selectedShape()`:

1. Получить выбранную фигуру в активном PowerPoint.
2. Если фигура не выбрана — показать сообщение “No shape selected”.
3. Если выбрано несколько фигур, использовать первую выбранную фигуру или показать предупреждение, в зависимости от того, что проще реализовать стабильно.
4. Прочитать все Tags выбранной фигуры.
5. Отобразить Tags в таблице.
6. Операции Add/Edit/Delete должны применяться к Tags выбранной фигуры.

## Операции с Tags

### Add Tag

При добавлении Tag:

1. Открыть диалог или inline editor с полями:

   * Name
   * Value
2. Проверить, что Name не пустой.
3. Если Tag с таким Name уже существует, спросить подтверждение на overwrite или выполнить update, если библиотека не поддерживает дубликаты.
4. Сохранить Tag в PowerPoint.
5. Обновить таблицу.

### Edit Tag

При редактировании Tag:

1. Пользователь выбирает строку в таблице.
2. Можно изменить Value.
3. Если технически возможно безопасно изменить Name — разрешить изменение Name через remove old + add new.
4. Сохранить изменение в PowerPoint.
5. Обновить таблицу.

### Delete Tag

При удалении Tag:

1. Пользователь выбирает строку в таблице.
2. Показать confirmation dialog.
3. Удалить Tag из PowerPoint.
4. Обновить таблицу.

### Refresh

Кнопка Refresh должна заново получить текущий объект PowerPoint согласно выбранной опции и перечитать Tags.

Это важно, потому что пользователь может переключать активный слайд или выбранную фигуру непосредственно в PowerPoint.

## Архитектура

Разделить код минимум на следующие логические части:

### PowerPointService

Отвечает за работу с `node-office-script`.

Методы:

* `connectToActivePresentation()`
* `getActiveSlide()`
* `getSelectedShape()`
* `getTags(target)`
* `setTag(target, name, value)`
* `removeTag(target, name)`
* `refreshTarget(mode)`

### UI Layer

Отвечает за:

* выбор режима;
* отображение таблицы;
* диалоги добавления/редактирования;
* сообщения об ошибках;
* блокировку кнопок, если объект недоступен.

### TagEditorState

Хранит текущее состояние:

* currentMode: `activeSlide | selectedShape`
* currentTarget
* currentTags
* connectionStatus
* lastError

## Требования к UX

UI должен быть максимально простой:

* одна форма;
* radio buttons для выбора режима;
* таблица Tags;
* кнопки Add/Edit/Delete/Refresh;
* статусная строка внизу окна.

В статусной строке показывать:

* connected/not connected;
* имя активной презентации, если доступно;
* текущий режим;
* ошибку, если она есть.

Пример:

`Connected: Demo.pptx | Mode: Active Slide | Tags: 3`

или:

`Error: No shape selected`

## Обработка ошибок

Обязательно обработать сценарии:

* PowerPoint не запущен.
* Нет активной презентации.
* Нет активного слайда.
* Нет выбранной фигуры.
* Ошибка чтения Tags.
* Ошибка записи Tag.
* Ошибка удаления Tag.
* Пользователь переключил презентацию/слайд/shape в PowerPoint во время работы приложения.

Ошибки не должны приводить к падению приложения. Показывать их в UI.

## Важные технические детали

Перед реализацией изучить API `Miramac/node-office-script`, особенно:

* получение активной презентации;
* `presentation.activeSlide()`;
* `presentation.selectedShape()`;
* API доступа к `.tags`;
* методы `.tag.get(name)`, `.tag.set(name, value)`, `.tag.remove(name)` или их актуальные аналоги.

Если API библиотеки отличается от ожидаемого, адаптировать код под фактический API библиотеки, но сохранить описанное поведение приложения.

## Acceptance Criteria

Приложение считается готовым, если:

1. Запущенный PowerPoint с открытой презентацией определяется приложением.
2. Приложение показывает Tags активного слайда.
3. Приложение показывает Tags выбранной фигуры.
4. Можно добавить Tag к активному слайду.
5. Можно изменить Tag активного слайда.
6. Можно удалить Tag активного слайда.
7. Можно добавить Tag к выбранной фигуре.
8. Можно изменить Tag выбранной фигуры.
9. Можно удалить Tag выбранной фигуры.
10. После изменений PowerPoint реально содержит обновлённые Tags.
11. Refresh корректно перечитывает текущий активный слайд или выбранную фигуру.
12. При отсутствии PowerPoint/презентации/выбранной фигуры приложение показывает понятную ошибку и не падает.
